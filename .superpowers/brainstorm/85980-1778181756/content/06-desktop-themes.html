What about the desktop (Swing) app?

The desktop app currently reads themes.json from platform-core to apply FlatLaf themes. If we delete themes.json, the desktop app loses its theme data. These are two completely different rendering engines — DaisyUI themes don't apply to Swing.

A

Desktop keeps its own theme list (Recommended)

Delete themes.json from platform-core. Desktop app hardcodes its own 5-6 FlatLaf-compatible theme definitions (dark, light, etc.). Web and desktop have independent theme systems — they don't need to match 1:1.

B

Keep themes.json for desktop compatibility

Keep themes.json in platform-core but web ignores it entirely (uses DaisyUI themes instead). Desktop still reads it for FlatLaf. Shared artifact but only one consumer.
