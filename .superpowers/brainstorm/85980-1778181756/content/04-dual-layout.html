Keep both layout modes or simplify?

You currently support sidebar and topbar layouts, switchable per user preference. DaisyUI's drawer handles the sidebar natively. Maintaining both means two layout templates that must stay in sync.

A

Keep both — sidebar + topbar

Keep the user-selectable layout preference. DaisyUI drawer for sidebar mode, DaisyUI navbar for topbar mode. LayoutRouter.kte stays. Two templates to maintain, but users get their choice.

B

Sidebar only — drop topbar layout

Remove the topbar layout and LayoutRouter.kte. One layout template using DaisyUI drawer. Remove the layout selector from settings. Simpler codebase, fewer templates to maintain.
