How should theme selection work in the UI?

DaisyUI has 30+ built-in themes. Users currently pick from a dropdown in the sidebar. With DaisyUI, you could show actual color previews instead of just names.

A

Simple select dropdown (current pattern)

Keep the sidebar dropdown selector. Replace themes.json data with DaisyUI theme names. Theme applies instantly via data-theme attribute swap. No visual preview — just the name. Works well in the compact sidebar.

B

Color swatch grid on settings page

Move theme selection to the Settings > Appearance tab. Show a grid of theme names with colored accent dots or mini-preview bars. Sidebar selector removed. Users go to settings to change themes — more room for preview but extra click.

C

Both — dropdown in sidebar, preview in settings

Quick switch via sidebar dropdown (names only). Full preview grid in Settings > Appearance for browsing. Same DaisyUI theme list used in both places.
